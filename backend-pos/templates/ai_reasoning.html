Langkah Demi Langkah Logika Model

1

AI menarik log historis transaksi dari database untuk menjumlahkan volume penjualan menu dan margin keuntungan bersihnya. Skala data tersebut disetarakan menggunakan normalisasi data terkecil dan terbesar (MinMax) agar selisih harga tidak menutupi volume penjualan.

2

Algoritma menetapkan 3 pusat koordinat acak (*Centroids*), kemudian menghitung jarak Euclidean dari setiap produk ke 3 pusat tersebut, lalu berulang kali menyesuaikan posisinya hingga konvergen (tidak bergeser lagi).

3

Setelah selesai dikelompokkan, AI menghitung total penjumlahan centroid. Klaster dengan koordinat centroid tertinggi dilabeli 'Laris & Untung Besar' (bintang utama), klaster terendah dilabeli 'Kurang Laris', dan sisanya dilabeli 'Menengah'. Data ini langsung diperbarui di database untuk disajikan ke admin.

Linear Regression

Prediksi Stok Sisa 7 Hari

Menganalisis tren pemakaian bahan baku harian dari data log inventaris dan melakukan proyeksi kebutuhan bahan untuk 7 hari ke depan guna mengantisipasi keterlambatan restock.

Persamaan Tren Matematis

Formula Garis Regresi Linier

y = mx + c

Dimana $y$ adalah jumlah pemakaian bahan baku (stock_used), $x$ adalah indeks hari berurutan, $m$ adalah kemiringan tren (slope), dan $c$ adalah intercept dasar.

Metrik Pengujian Error

MAPE (Mean Absolute Percentage Error)

AI mengevaluasi ketepatan model dengan menghitung persentase rata-rata penyimpangan prediksi terhadap pemakaian aktual masa lalu. Nilai MAPE ini disimpan per bahan baku di tabel ai_prediction_results untuk memberikan indikasi tingkat kepercayaan prediksi.

Visualisasi Alur Algoritma (Data Pipeline)

1. ETL Data Log

Mengekstrak riwayat pemakaian harian (`stock_used`) dari `inventory_logs`

→
2. Per Bahan Baku

Mengelompokkan data historis terpisah per bahan baku (minimal 5 hari terakhir)

→
3. Latih Tren

Model regresi ditarik melewati sebaran data pemakaian aktual ($x$ = hari, $y$ = terpakai)

↓
6. Supabase DB

Menyimpan nilai ramalan sisa stok & MAPE ke tabel `ai_prediction_results`

←
5. Kalkulasi Sisa Stok

Mengurangi stok saat ini dengan total prediksi konsumsi 7 hari kedepan (bisa minus)

←
4. Proyeksi 7 Hari

Meramalkan konsumsi kumulatif untuk rentang hari ke-[N+1] s/d [N+7]

Langkah Demi Langkah Logika Model

1

Algoritma memisahkan riwayat log inventaris per bahan baku. Syarat training model berjalan adalah minimal tersedianya histori data pemakaian selama 5 hari ke belakang.

2

Model dilatih menggunakan regresi linier satu dimensi untuk memetakan arah garis tren pemakaian bahan baku per hari. Dari situ diperoleh koefisien gradien ($m$) yang mengindikasikan apakah konsumsi bahan semakin menurun, stabil, atau menanjak tajam.

3

Model meramalkan pemakaian untuk 7 hari mendatang, kemudian menjumlahkannya. Sisa stok 7 hari mendatang dihitung dengan rumus: Stok Saat Ini - Total Prediksi Konsumsi 7 Hari. Jika hasil perhitungan bernilai **negatif (minus)**, itu adalah pertanda stok akan habis dan menginstruksikan admin untuk melakukan pengisian ulang bahan baku tersebut.

RAM N-Gram Suggestions

Rekomendasi Pencarian & Frekuensi Pembelian

Menyimpan skor keterpilihan menu di dalam memori RAM server Flask untuk memberikan rekomendasi pencarian (autocomplete) instan saat kasir mengetik nama menu di POS.

Teknologi Penyimpanan Kilat

RAM In-Memory Dictionary

RAM Cache

Pencarian autocomplete tidak menyentuh database disk melainkan dicari langsung di memori RAM melalui data terindeks untuk respon pencarian kurang dari 5 milidetik.

Aliran Transaksi Asinkron

Fire-and-Forget API Checkout

Sesaat setelah kasir mengklik tombol checkout, React frontend menembakkan API asinkron ke Flask tanpa menunggu responnya (tanpa await). Ini mencegah UI kasir lag dan skor frekuensi produk langsung dinaikkan di memori RAM.

→
2. Fire-and-Forget

Saat transaksi sukses di POS, React mengirim nama menu ke endpoint `/api/ngram/increment` secara asinkron

→
3. Increment RAM

Flask menambah skor keterpilihan menu sebesar +1 secara langsung di memori RAM server

↓
6. Kasir Autocomplete

POS menerima dan langsung menyajikan 5 rekomendasi terpopuler untuk mempermudah kasir mengetik

←
5. Sort & Slice

Hasil yang lolos diurutkan berdasarkan skor terbesar lalu diambil 5 menu teratas

←
4. Match Substring

Saat kasir mengetik di kolom pencarian POS, request dikirim ke endpoint `/api/suggest?q=...` untuk dicari di RAM

Langkah Demi Langkah Logika Model

1

Saat server pertama kali menyala, fungsi load_initial_data() memicu penarikan seluruh menu terdaftar dari database Supabase dan memuatnya ke dalam memori RAM server sebagai dictionary Python dengan inisialisasi skor 0.

2

Setiap kali transaksi checkout terjadi di halaman POS kasir, data keranjang dikirim ke endpoint `/api/ngram/increment`. Flask langsung menaikkan skor produk tersebut di RAM tanpa menyentuh database disk, menjamin proses transaksi berjalan sangat kilat.

3

Saat kasir mengetik di pencarian POS kasir, request asinkron dikirim ke /api/suggest?q=.... Flask melakukan pencarian substring di RAM secara case-insensitive, mengurutkannya berdasarkan produk dengan skor keterpilihan tertinggi, dan mengembalikan 5 nama menu teratas.
